## Architecture & Design
1. Ensure each agent (Planner, Researcher, Reasoner, Critic, Compressor) has a single, clear responsibility.
2. Use dependency injection for model, storage, and web adapters to improve testability.
3. Keep all configuration in config.py; avoid hard-coded values elsewhere.
4. Document all public interfaces with clear input/output/failure docstrings.
5. Use Python type hints throughout for clarity and static analysis.
6. Prefer immutable dataclasses (frozen=True) for value objects.
7. Use __slots__ in dataclasses to reduce memory overhead.
8. Validate all dataclass inputs at construction or via validate() methods.
9. Separate orchestration logic from agent logic in orchestrator.py.
10. Keep the dashboard as a read-only event consumer, not a state owner.

## Resource Management
11. Enforce VRAM and RAM limits at runtime; fail gracefully if exceeded.
12. Use asyncio.Semaphore to limit concurrent model and embedding jobs.
13. Unload or release model sessions after idle timeout.
14. Bound all queues to prevent unbounded memory growth.
15. Add resource usage monitoring (RAM, VRAM) to the dashboard.

## Persistence & Storage
16. Use SQLite for all persistent storage; avoid in-memory-only state.
17. Store append-only logs as JSONL for traceability.
18. Separate tables for vectors, source docs, macros, logs, and metrics.
19. Avoid re-indexing the full corpus on every startup.
20. Store provenance and source metadata with all evidence.

## Retrieval & Embeddings
21. Use Chroma PersistentClient as the default vector store.
22. Support asymmetric query/document encoding if the model allows.
23. Add metadata filters and reranking to retrieval.
24. Use FTS5 or similar for lexical candidate generation.
25. Avoid loading all vectors into memory for each search.

## Macro Engine & Compression
26. Represent reasoning as a graph/DAG, not just token lists.
27. Use a compact, typed opcode lexicon for core operations.
28. Store macro registry and decoder lexicon in machine-readable form.
29. Add proof-hash and round-trip verification for compressed traces.
30. Limit macro recursion depth (e.g., 8).

## Agents & Pipeline
31. Pass ResourceBudget through all agent calls.
32. Make thinking_minutes affect actual work depth, not just metadata.
33. Use schema-constrained JSON for all agent outputs.
34. Validate model outputs before passing to the next stage.
35. Allow one repair attempt for invalid outputs, then fail structurally.

## Testing & Validation
36. Write unit tests for every agent with mocked dependencies.
37. Add end-to-end pipeline tests for at least two example questions.
38. Test both fast and deep reasoning modes.
39. Add tests for resource limits, fallback, and degraded modes.
40. Add regression tests for old serialized payloads.

## Error Handling & Robustness
41. Define explicit error types for startup, backend, timeout, and OOM.
42. Add cancellation and retry logic for orchestrator tasks.
43. Never retry invalid outputs indefinitely.
44. Surface warnings and degraded-mode events in the dashboard.
45. Fail clearly when required dependencies or models are missing.

## Dashboard & UI
46. Use Tkinter for the default GUI to minimize memory use.
47. Run model and I/O work outside the main UI thread.
48. Show live status updates using a bounded queue.
49. Display resource usage and error states clearly.
50. Show when events are dropped due to queue backpressure.

## Optimization & Self-Improvement

52. Persist optimizer proposals and activation decisions.
53. Cap history windows to avoid memory bloat.
54. Export only verified deep-mode traces for future training.
55. Never let the optimizer apply live changes without validation.
